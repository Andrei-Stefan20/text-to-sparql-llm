# DPO Fine-Tuning Dataset Management

## Overview
This directory contains the dataset preparation and fine-tuning infrastructure for Direct Preference Optimization (DPO) of SPARQL generation models.

## Structure

```
dpo_finetuning/
├── datasets/
│   ├── raw/                 # Raw collected data from few-shot errors
│   └── processed/           # Processed DPO pairs (prompt, chosen, rejected)
├── scripts/                 # Data processing and training scripts
├── configs/                 # Training configuration files
└── README.md
```

## DPO Pair Format

Each training pair should follow this JSON format:

```json
{
  "prompt": "Natural language question",
  "chosen": "Gold standard SPARQL query",
  "rejected": "Incorrect SPARQL query generated by few-shot",
  "question_id": "unique_identifier",
  "metadata": {
    "source": "few-shot-error|manual|decomposition",
    "error_type": "syntax|semantic|limit_placement",
    "timestamp": "2025-12-06T12:00:00"
  }
}
```

## Files

- **raw/**: Directory containing raw data collection
  - `few_shot_errors.jsonl` - Incorrect queries from few-shot generation
  - `gold_standard.jsonl` - Correct reference queries
  
- **processed/**: Processed DPO training data
  - `dpo_pairs.jsonl` - Final DPO pairs for training
  - `train_split.jsonl` - Training set
  - `validation_split.jsonl` - Validation set

## Usage

1. **Collect Data**: Store few-shot errors in `raw/few_shot_errors.jsonl`
2. **Prepare Gold Standard**: Add correct queries to `raw/gold_standard.jsonl`
3. **Process**: Run `scripts/prepare_dpo_dataset.py` to create pairs
4. **Train**: Use `configs/dpo_config.yaml` for training setup
